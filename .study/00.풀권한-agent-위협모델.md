
# 풀권한 Agent의 근본적 문제

- **프롬프트 인젝션 너무 쉽고 위협적이다**
    - agent 특성상 일반 텍스트도 실행해야 하는 명령어로 인식할 수 있다
    - 그냥 agent가 참고할만한 아무 경로에 뿌려놓으면 알아서 줏어간다
    - agent가 참조가능한 모든 소스를 조심해야 함. 현실적으로 말이 안됨
    - 특히 장기 메모리 쓰는 경우 해킹컨텍스트 한번만 잘 심어두면 국밥임
    - 누가 어떤 경로로 심은건지 + 심지어 언제 당한건지도 알아채기 어려움
- **agent만 털면 내 모든 권한 다 털림**
    - 원래는 github / wiki / 네이버메일 하나하나 뚫어야 함
- **보안 설정 대충하게 됨**
    - 보안 빡세게 잡으려면 설정이 복잡해질 수 밖에 없음
    - but, 문화상 설정 어려우면 아무도 안씀
- **일개 개발자들이 만든 코드 어떻게 신뢰?**
    - 일개 개발자가 만든걸 많은 사람이 쓰는 생태계라 그 자체가 위험
    - openclaw의 경우, 코드 자체 결함으로 tool call로 OS레벨 cli 같은거 호출하는데서 온갖 취약점 발견됨
    - 특히, skill이나 플러그인 마켓플레이스는 더더욱 이 문제가 심각
